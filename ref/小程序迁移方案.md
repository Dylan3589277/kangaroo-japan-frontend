## 二、迁移策略

### 核心原则

1. **优先迁移代拍核心流程**：Mercari/Yahoo/Amazon 三大平台的商品浏览+详情
2. **复用独立站现有架构**：shadcn/ui组件、zustand状态管理、next-intl国际化
3. **后端API已就绪**：NestJS后端已有对应的API模块（yahoo竞拍、签到等）
4. **小程序逻辑照搬**：相同的API调用逻辑和UI布局，用Next.js重写

### 分阶段推进

| 阶段        | 内容                                  | 工作量估计 |
| ----------- | ------------------------------------- | ---------- |
| **Phase 1** | Mercari商品列表+详情页                | 3-4个页面  |
| **Phase 2** | Yahoo商品列表+详情+出价页             | 4-5个页面  |
| **Phase 3** | Amazon商品列表+详情页                 | 3-4个页面  |
| **Phase 4** | 用户中心（竞拍记录/押金/会员/优惠券） | 6-8个页面  |
| **Phase 5** | 仓库管理                              | 10个页面   |
| **Phase 6** | 零散功能（签到/社区/消息/活动等）     | 10个页面   |
